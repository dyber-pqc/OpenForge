# Pin Planner

The Pin Planner provides a graphical interface for assigning top-level design ports to physical FPGA pins. It displays the target device's package diagram and lets you drag signals onto pins or assign them from a dropdown menu.

## Overview

The Pin Planner is available when your project targets an FPGA device. Open it from **View > Pin Planner** or from the FPGA section of the Flow Navigator.

The panel has two main areas:

1. **Signal list** (left): All unassigned and assigned top-level ports from your design
2. **Package diagram** (right): Interactive view of the FPGA package with pin labels

## Assigning Pins

### Drag and Drop

1. Select a signal name in the signal list
2. Drag it onto a pin in the package diagram
3. The pin changes color to indicate the assignment:
    - Blue = input
    - Green = output
    - Yellow = bidirectional
    - Gray = unassigned

### Right-Click Assignment

1. Right-click a pin in the package diagram
2. Select **Assign Signal** from the context menu
3. Choose a signal from the dropdown list
4. Click **OK**

### Table Entry

The signal list also works as an assignment table:

| Column | Description |
|---|---|
| Signal | Port name from your RTL |
| Direction | Input / Output / Inout |
| Pin | Assigned package pin (editable) |
| I/O Standard | Voltage standard (LVCMOS33, LVDS, etc.) |
| Drive | Drive strength in mA |
| Pull | Pull-up / pull-down configuration |

Double-click the **Pin** column to type a pin number directly.

## Pin Types

The package diagram color-codes pins by function:

| Color | Type | Description |
|---|---|---|
| White | GPIO | General-purpose I/O |
| Blue | Clock | Dedicated clock input pins |
| Red | Power | VCC / VDD supply pins |
| Black | Ground | GND pins |
| Yellow | Config | Configuration pins (JTAG, SPI flash) |
| Gray | NC | No-connect pins |

## Constraint Export

After assigning pins, export the constraint file:

1. Click **Export Constraints** in the toolbar
2. Choose the format:
    - **PCF** for iCE40 targets
    - **LPF** for ECP5 targets
    - **XDC** for Xilinx targets
3. Select the output file location
4. The constraint file is written and automatically added to your project

### Generated PCF Example

```
# Auto-generated by OpenForge Pin Planner
set_io clk 35
set_io rst_n 2
set_io led[0] 11
set_io led[1] 37
set_io led[2] 31
set_io spi_mosi 14
set_io spi_miso 17
set_io spi_sck 15
```

## Conflict Detection

The Pin Planner validates assignments in real time:

| Conflict | Description |
|---|---|
| Duplicate pin | Two signals assigned to the same pin |
| Invalid pin | Signal assigned to a power, ground, or NC pin |
| I/O bank conflict | Mixed voltage standards in the same I/O bank |
| Clock routing | Non-clock signal on a dedicated clock pin (warning) |

Conflicts are highlighted in red in both the signal list and the package diagram.

## Keyboard Shortcuts

| Shortcut | Action |
|---|---|
| `Ctrl+Shift+N` | Open Pin Planner |
| `Delete` | Unassign the selected pin |
| `Ctrl+F` | Search pins by name or number |
| `+` / `-` | Zoom the package diagram |
| `F` | Fit the package diagram to the panel |
| `Ctrl+E` | Export constraints |

## Supported Devices

The Pin Planner includes package data for:

| Family | Packages |
|---|---|
| iCE40 LP/HX | QFN48, QFN84, BGA256 |
| iCE40 UP5K | SG48, UWG30 |
| ECP5 | CABGA256, CABGA381, CABGA554 |
| Gowin | QFN48, QFN88, BGA256 |

!!! note "Adding custom devices"
    Device package data is stored as JSON files in `share/devices/`. You can add custom device definitions by creating a JSON file with pin names, coordinates, and types.

## Tips

!!! tip "Start with clocks"
    Assign clock signals first -- they must go on dedicated clock pins for best performance. The Pin Planner highlights clock-capable pins.

!!! tip "Check the board schematic"
    Your pin assignments must match the PCB connections on your development board. Keep the board schematic open alongside the Pin Planner.

!!! tip "I/O banks"
    Group signals by voltage standard. Mixing LVCMOS33 and LVCMOS18 in the same I/O bank can cause electrical issues on some FPGA families.
